# Architecture & Learning Outcomes

_This file will hold the full architecture diagram once Stage 3+ introduces
the networking topology worth diagramming. For now, it tracks the learning
outcomes this project is designed to cover._

## Learning outcomes map

- **Compute**: container orchestration (AKS), autoscaling, VM basics
- **Networking**: VNet/subnetting, NSGs, load balancing (L4 vs L7), DNS,
  Kubernetes networking (CNI, Services, Ingress, NetworkPolicy), TLS
- **Storage & data**: managed Postgres, connection pooling, indexing,
  Kubernetes persistent storage
- **Identity & security**: managed identities, Key Vault, Azure RBAC vs
  Kubernetes RBAC, image scanning
- **Deployment & release engineering**: rolling/blue-green/canary, CI/CD,
  Infrastructure as Code
- **Observability & reliability**: metrics, tracing, health checks,
  resilience patterns
- **Cost & governance**: tagging, budgets, right-sizing

Each item gets checked off with a link to the relevant ADR and code as the
stages are completed.
